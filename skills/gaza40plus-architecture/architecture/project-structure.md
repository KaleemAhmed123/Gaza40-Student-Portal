# Gaza40Plus Project Structure

This document outlines the top-level directory structure of the `Gaza40Plus` frontend repository, explaining the purpose of each directory and identifying patterns for reusable code, feature modules, and files that are rarely modified.

## Top-Level Directory Layout

The repository is built as a **Next.js** application using the **App Router** pattern inside the `src` directory.

### Root Directory
* `src/`: The main application source directory.
* `public/`: Static assets (images, logos, fonts, icons) served directly by Next.js.
* `plans/`: Local documentation, roadmaps, and implementation plans.
* `node_modules/`: Package manager dependencies.
* `.next/`: Automated compilation and build outputs generated by Next.js.

---

## Detailed Directory Breakdown (under `src/`)

The `src/` directory contains the source code for the application and is structured as follows:

| Directory | Purpose | Classification | Modifiability |
| :--- | :--- | :--- | :--- |
| `src/app` | Implements Next.js App Router folders representing URL routes and layouts. | Feature Modules | **Frequently Modified** (when adding/editing routes) |
| `src/components` | Houses components. Subdivided into `ui/` for basic widgets and `shared/` for page-level structural blocks. | Reusable Code | **Frequently Modified** |
| `src/context` | Context providers (e.g., `SocketContext.tsx`) for low-level react state/events. | Reusable Code / State | **Rarely Modified** (once configuration is set) |
| `src/hooks` | Custom React hooks (e.g., `useTranslation.ts`, `useUrlSelector.ts`) used across files. | Reusable Code | **Occasionally Modified** |
| `src/lib` | Configuration files, helpers, API adapters, and general utility functions. | Reusable Code | **Occasionally Modified** |
| `src/locales` | Static translation files (e.g., `ar.ts`, `en.ts`) defining translation keys. | Localization | **Rarely Modified** (except translation edits) |
| `src/redux` | Redux store configuration and slices (`redux/slices/*`) for global client state. | State Management | Slices: **Frequently**; Store/Boilerplate: **Rarely** |
| `src/services` | Service class layers that handle backend API requests (e.g. `axiosInstance.ts`, services/api/*). | Reusable Code | **Occasionally Modified** (upon API updates) |

---

## Folder Classifications

### 1. Reusable Code Folders
These folders contain generic, stateless, or general-purpose logic and UI modules that are consumed by multiple feature pages.
* **`src/components/ui/`**: Low-level interactive elements (e.g., buttons, inputs, document viewers).
* **`src/components/shared/`**: Reusable complex UI blocks (e.g., sidebars, navigation bars, headers).
* **`src/hooks/`**: Global react utilities that share stateful lifecycle behaviors.
* **`src/lib/`**: Generic JS/TS helper functions, theme constants, and adapters.
* **`src/services/`**: Standard Axios wrapper and endpoint-specific clients.

### 2. Feature Modules
These folders contain code tied directly to specific application domains, routes, or workflows.
* **`src/app/`**: Next.js App Router subdirectories corresponding to user flows:
  * `admin/`: Admin panels and settings.
  * `chat/`: Chat interface.
  * `forgot-password/` & `reset-password/`: Credential flows.
  * `login/` & `signup/` & `verify-email/`: Authentication flows.
  * `mentor/`: Mentor dashboard and associated workspace.
  * `onboarding/`: Initial user registration profile setup.
  * `regional-admin/`: Regional administration dashboards.
  * `student/`: Student portal, courses, and settings.
* **`src/redux/slices/`**: Redux state slices partitioned by domain (e.g., `authSlice`, `profileSlice`, `announcementsSlice`, `offersSlice`, `queriesSlice`, `uiSlice`).

### 3. Rarely Modified Folders
These folders contain boilerplate configurations, compiled assets, or translations that are updated only under specific conditions.
* **`src/locales/`**: Contain translation catalogs. Only modified to add new localization strings.
* **`src/context/`**: Global setup contexts (like socket connection wrappers) which are established during initial setup.
* **`src/redux/store.ts`**, **`src/redux/providers.tsx`**, and **`src/redux/hooks.ts`**: Setup boilerplate for Redux.
* **Root config files**: `.gitignore`, `tsconfig.json`, `next.config.ts`, `eslint.config.mjs`, `postcss.config.mjs`.
* **Build/dependency folders**: `.next/`, `node_modules/`, `.git/`.
